# hit-and-hit

https://alpacahack.com/daily/challenges/hit-and-hit

## 問題の概要

過去の問題 [hit-and-miss](https://alpacahack.com/daily/challenges/hit-and-miss) では、サーバーが入力した正規表現にフラグがマッチするかどうかを判定して「Hit!」か「Miss...」を答えてくれていました。

今回は、サーバーが入力した正規表現にフラグがマッチするかどうかを評価して、どちらの場合も「Hit!」と返すようになっています。

## 解法

マッチするときには短時間で評価が終わるが、マッチしないときには評価に時間がかかる正規表現を作成すれば、サーバーからのレスポンスの時間を測ることで、フラグが正規表現にマッチするかどうかを判定できます。

例えば、`^(\w+)+$` という正規表現で `aaaaaaaaaaaaaaaaaaaaa@` を評価すると、最後の `@` がマッチしないため `\w+` の分割方法が総当たりで試されることになり、文字の長さに応じて評価にかかる時間が指数関数的に増加します。

このような重い正規表現 `heavy_regex` を使って `"^{regex}|^{heavy_regex}$"` のように組み合わせることで、`regex` にマッチするときは短時間で評価が終わり、マッチしないときは重い正規表現の評価に時間がかかるようになります。

評価時間はサーバーの処理速度やフラグの長さで変わるため、実際のサーバーでテストしてちょうどいい重さの正規表現を探しました。

評価時間を長くすればマッチしているかどうかを区別しやすくなりますが、あまり長くすると探索に時間がかかりすぎます。

今回は `Alpaca\{((\w+)+)+$` を使用し、0.6 秒以上かかるときはマッチしていないと判断することにしました。
