# Compressor

https://alpacahack.com/daily/challenges/compressor

## 問題の概要

サーバーに文字列を渡すと フラグ + 入力した文字列 を zlib で圧縮した長さを教えてくれるので、その情報をもとにフラグを推測する問題です。

## 解法

フラグの分かっている文字列に次の候補となる 1 文字を足して入力し、圧縮後の長さが最も短くなる文字をフラグの次の文字と推測します。

zlib は同じ文字列のパターンが繰り返されると圧縮率が上がるため、フラグの次の文字が正しいときは他と比べて圧縮後の長さが短くなります。

これをフラグの最後まで繰り返すことでフラグ全体を推測することができます。

## 解答に使用したコード

フラグの文字列の出現回数が少ないと同じ長さになる候補が複数出ることがあるため、候補のパターンを何回か繰り返すようにしています。
