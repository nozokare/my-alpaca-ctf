# Optimized expression

https://alpacahack.com/daily/challenges/optimized-expression

## 問題の概要

入力されたフラグが正しいかどうかを判定するフラグチェッカーのバイナリとソースコードが与えられています。

判定の主要部分は次のようになっています。

```c:chal.c
for (int i = 0; i < TABLE_SIZE; ++i)
{
    uint64_t x = (uint8_t)buf[i];
    uint64_t y = (x * 3435973837) >> 34;
    uint64_t t = (x * 613566757) >> 32;
    uint64_t z = x - ((((x - t) >> 1) + t) >> 2) * 7;

    if (y != table[i][0] || z != table[i][1])
    {
        return 1;
    }
}
```

## 解法

処理内容の意味が分からなくても、各バイトの処理は独立しているため、1バイトずつ正しい値を求めていけばフラグが得られます。

```python:brute_force.py
table = [
    [13, 2],
    [21, 3],
    .....
]

def is_correct_byte(x, exptected):
    y = (x * 3435973837) >> 34
    t = (x * 613566757) >> 32
    z = x - ((((x - t) >> 1) + t) >> 2) * 7
    return y == exptected[0] and z == exptected[1]

buf = []
for expected in table:
    for x in range(256):
        if is_correct_byte(x, expected):
            buf.append(x)
            break

print(bytes(buf))
```

## 処理の内容は何なのか

入力 `x` に対して `y`, `t`, `z` の値を計算してみましょう。

```python
for x in range(32):
    y = (x * 3435973837) >> 34
    t = (x * 613566757) >> 32
    z = x - ((((x - t) >> 1) + t) >> 2) * 7
    print(f"{x}, {y}, {t}, {z}")
# =>
# 0, 0, 0, 0
# 1, 0, 0, 1
# 2, 0, 0, 2
# 3, 0, 0, 3
# 4, 0, 0, 4
# 5, 1, 0, 5
# 6, 1, 0, 6
# 7, 1, 1, 0
# 8, 1, 1, 1
# 9, 1, 1, 2
# 10, 2, 1, 3
# 11, 2, 1, 4
# 12, 2, 1, 5
# 13, 2, 1, 6
# 14, 2, 2, 0
# ...
# 28, 5, 4, 0
# 29, 5, 4, 1
# 30, 6, 4, 2
# 31, 6, 4, 3
```

`y = x // 5`, `t = x // 7`, `z = x % 7` になっています。

`y = (x * 3435973837) >> 34` のような式は、定数除算を逆数の掛け算で高速化したもので、コンパイラの最適化によってよく生成されるコードです。

`3435973837` は　`2**34 / 5` を計算した値で、`1/5` を固定小数点数で表したものとなっています。

```python
(1 << 34) / 5  # => 3435973836.8
```

CPU の除算命令は掛け算やシフトに比べて遅いことが多いため、定数で割る場合はこのような方法で高速化されることがよくあります。

`z = x - ((((x - t) >> 1) + t) >> 2) * 7` については、`t = x // 7` として詳しく計算すると `z = x - (x // 7) * 7` と同じ意味になっており、`z = x % 7` を計算していることになります。

## 解法(2)

`y` と `z` が、`x` を 5 で割った商と 7 で割った余りを表していることが分かったので、`y` と `z` から直接 `x` を求めることもできます。

```python
table = [
    [13, 2],
    [21, 3],
    .....
]

flag = [y * 5 + ((z - y * 5) % 7) for [y, z] in table]
print(bytes(flag))
```

Python の剰余は常に正の数を返すため、`(z - y * 5) % 7` とすることで 7 で割った余りを補正できます。
