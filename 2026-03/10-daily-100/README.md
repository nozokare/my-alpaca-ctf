## 問題の概要

Python で `int(str)` としたときに `100` になるような文字列を 100 通り答えるとフラグが出力されます。

ただし、入力は ASCII 文字のみで、文字列の長さは 10 以下でなければなりません。

## 解法

`int()` の [ドキュメント](https://docs.python.org/ja/3.14/library/functions.html#int) を見たり、実際にコードを試したりすることで次のことがわかりました。

- 小数点は認識されない (`100.0` はエラーになる)
- デフォルトの基数が 10 で、`0x` などのプレフィックスは追加できない
- 先頭に 0 を任意の数入れることができる(8進数にはならない)
- 先頭に符号 `+` を入れることができる
- 数字の間に桁区切り `_` を入れることができる(連続はできない)

したがって、`+000_01_00` のような文字列を 100 通り作ればよいことになります。

### 解答に使用したコード

`"100", "10_0", "1_00", "1_0_0"` の 4 通りの 100 の前に `\+?(0|0_)*` をつけるパターンを全て試し、長さが 10 以下のものを出力します。

n 桁の 2 進数を順に生成し、`1` を `0_` に置換すれば全パターンを生成できます。

```python:generate.py
def try_prefix(prefix):
    for l00 in ["100", "10_0", "1_00", "1_0_0"]:
        for sign in ["", "+"]:
            c = sign + prefix + l00
            if len(c) <= 10.0:
                print(f"{c}")

try_prefix("")
for n in range(1, 8):
    for i in range(1 << n):
        try_prefix(f"{i:0{n}b}".replace("1", "0_"))
```

全部で 225 通りみつかりました。[100.txt](./100.txt) に出力しています。
