# bloom

https://alpacahack.com/daily/challenges/bloom

## 問題の概要

`\n` を入力するとフラグとランダムなバイト列の XOR を取った hex string が返ってくる Python サーバーが動いており、受け取ったデータからフラグを復号する問題です。

## 問題の分析

```python
# it is so secure randint function!!
def secure_randint(a, b):
    return secrets.randbelow(b - a + 1) + a
```

暗号学的に安全な乱数生成器が使用されているため、これは本当に安全なランダムな整数を生成します。

完全にランダムなバイト列と XOR を取ると、AES で暗号化しているようなもので、フラグを復元することはできません。

しかし実際は、

```python
key = bytes([secure_randint(1, 255) for _ in range(len(plain))])
```

のように `1` から `255` までの整数を生成しており、偏りのある乱数列を使用していることがわかります。

## 解法

FLAG の各ビットは 128/255 の確率で反転し、127/255 の確率で反転しません。

大量のサンプルを取得し、ビット毎に `0` と `1` のどちらが多いか統計をとることで元のビットを推定できそうです。

配布プログラムをベースにしてローカルで試してみたのですが、10万サンプルほどでは一部のビットが化けて完全に復元できず、100万サンプルほどで完全に復元できました。

問題サーバーがどれくらいの速度でサンプルを返してくれるかによっては別の方針が必要になるかもしれません。

サーバーの負荷が心配ですが、適切に throttle が設定されているイメージなので、試しに1～2分ほどサンプルを取得してみます。

```bash
yes "\n" | nc ........ | pv | cut -b57- > flag.hex
```

1MB/s 程度の速度でダウンロードでき、2 分で 88 万サンプルほど取得できました。

一度サンプルを取得してしまえばあとはローカルで解析できます。

## 正攻法について

他の方の解法を見てみると、バイトごとに 0〜255 の値のうち出現しなかった値を探すのが正攻法のようです。
